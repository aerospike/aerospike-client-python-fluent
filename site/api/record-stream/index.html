    RecordStream¶

    RecordStream ¶

    RecordStream(source)

    Async iterator of :class:~aerospike_fluent.record_result.RecordResult rows.

    Produced by await session.query(...).execute() and similar APIs. Prefer async for row in stream, or helpers such as :meth:collect and :meth:first. Do not call RecordStream(...) directly; use factories like :meth:from_list or :meth:from_batch_records.

    Example

    Typical consumption with async for::

    stream = await session.query(key).bins(["name"]).execute()
async for row in stream:
    if row.is_ok and row.record:
        print(row.record.bins)
    See Also

    :meth:first_or_raise: Assert a single OK row.

    from_list classmethod ¶

    from_list(results)

    Wrap an already-materialised list of results.

    Example:: stream = RecordStream.from_list([row1, row2]) rows = await stream.collect()

    chain classmethod ¶

    chain(streams)

    Yield all results from each stream in order.

    Example:: combined = RecordStream.chain([stream_a, stream_b])

    from_batch_records classmethod ¶

    from_batch_records(batch_records)

    Wrap a sequence of async-client BatchRecord objects.

    Example:: stream = RecordStream.from_batch_records(batch_records)

    from_recordset classmethod ¶

    from_recordset(recordset)

    Wrap a Recordset (async iterable of Record).

    Each yielded Record is converted to a :class:RecordResult with result_code=OK and index=-1 (queries have no positional index).

    Example:: stream = RecordStream.from_recordset(recordset)

    from_single classmethod ¶

    from_single(key, record)

    Wrap a single-key result.

    Example:: stream = RecordStream.from_single(key, record)

    from_error classmethod ¶

    from_error(
    key, result_code, in_doubt=False, exception=None
)

    Wrap a single-key error as a one-element stream.

    Example:: stream = RecordStream.from_error(key, ResultCode.TIMEOUT)

    __anext__ async ¶

    __anext__()

    Yield the next :class:~aerospike_fluent.record_result.RecordResult.

    Raises:

    Type Description
    StopAsyncIteration

    When the stream is exhausted or :meth:close was called before more rows were read.

    first async ¶

    first()

    Consume and return the first row, or None if there are no rows.

    Returns:

    Type Description
    RecordResult | None

    The first :class:~aerospike_fluent.record_result.RecordResult, or
    RecordResult | None

    None when the stream is empty.
    Note

    This advances the iterator; remaining rows are left for further async for or other helpers only if the underlying source allows partial consumption (most fluent streams are single-pass).

    Example:: stream = await session.query(key).execute() row = await stream.first() if row is None: ...

    first_or_raise async ¶

    first_or_raise()

    Return the first row and require success (see :meth:RecordResult.or_raise).

    Returns:

    Type Description
    RecordResult

    The first OK :class:~aerospike_fluent.record_result.RecordResult.

    Raises:

    Type Description
    StopAsyncIteration

    If the stream yields no rows (empty).
    AerospikeError

    If the first row is not OK (from :meth:RecordResult.or_raise).
    Example

    rec = (await stream.first_or_raise()).record_or_raise()

    first_udf_result async ¶

    first_udf_result()

    Scan forward for the first non-None :attr:~aerospike_fluent.record_result.RecordResult.udf_result.

    Returns:

    Type Description
    Any | None

    The UDF return value, or None if no row carries a UDF result.

    Example:: value = await stream.first_udf_result()

    See Also

    :meth:Session.execute_udf: Produces streams with UDF results.

    collect async ¶

    collect()

    Drain the stream into a list (order preserved).

    Returns:

    Type Description
    list[RecordResult]

    All remaining :class:~aerospike_fluent.record_result.RecordResult
    list[RecordResult]

    instances.
    Example

    rows = await stream.collect() oks = [r for r in rows if r.is_ok]

    failures async ¶

    failures()

    Drain the stream and return rows where :attr:~aerospike_fluent.record_result.RecordResult.is_ok is false.

    Returns:

    Type Description
    list[RecordResult]

    Only error or non-OK rows.
    Note

    Like :meth:collect, this consumes the entire stream.

    close ¶

    close()

    Mark the stream closed; further :meth:__anext__ calls stop iteration.

    Idempotent. Use when abandoning a stream early to cooperate with resource cleanup where supported.

    SyncRecordStream¶

    SyncRecordStream ¶

    SyncRecordStream(stream, loop_manager)

    Blocking view of :class:~aerospike_fluent.record_stream.RecordStream.

    Iterator protocol and helpers (first, collect, …) call into the underlying async stream via the parent client's event-loop manager, so each step may block the calling thread until data is ready.

    Behavior vs async: Ordering and per-row semantics match :class:~aerospike_fluent.record_stream.RecordStream; only the API surface is synchronous. close() closes the async stream directly without crossing the loop (releases resources; pending async iteration should stop).

    Example:: with SyncFluentClient("localhost:3000") as client: session = client.create_session() for row in session.query(ns, set).bins(["name"]).execute(): if row.record: print(row.record.bins)

    See Also

    :class:~aerospike_fluent.record_stream.RecordStream

    Wrap stream for synchronous consumption (internal use).
